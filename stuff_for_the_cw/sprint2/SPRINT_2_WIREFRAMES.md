# SPRINT 2: WIREFRAMES
## Community Item Sharing Platform - All Required Pages

Wireframes show the layout and structure of each page without detailed design.

---

## REQUIRED PAGES (Per Brief)

1. Users List Page
2. User Profile Page
3. Listings Page (Browse)
4. Listing Detail Page
5. Categories/Tags Page

## ADDITIONAL ESSENTIAL PAGES

6. Homepage
7. Create Listing Form
8. Dashboard
9. Request Management Pages

## WIREFRAME 1: HOMEPAGE

### Purpose:
Welcome visitors, explain platform, encourage sign-up

### Layout Description:

**Header (all pages):**
- Logo (left)
- Navigation: Browse | Users | Categories | Log In | Sign Up (right)
- Search bar (center)

**Hero Section:**
- Large heading: "Share Items with Your Community"
- Subheading: "Borrow, swap, or give away items locally - no money needed"
- Two buttons: [Sign Up Free] [Browse Items]
- Background image: Community sharing items

**How It Works (3 columns):**
- Column 1: Icon + "List Your Items" + Brief description
- Column 2: Icon + "Browse & Request" + Brief description  
- Column 3: Icon + "Exchange & Build Trust" + Brief description

**Featured Listings (Grid):**
- Section title: "Recently Added Items"
- 4-6 listing cards (photo, title, category, exchange type, distance)
- [View All Items] button

**Categories Overview:**
- Section title: "Browse by Category"
- Category cards in grid (icon, name, count)
- Tools | Books | Baby Items | Sports | Household | Electronics

**Call to Action:**
- "Join 500+ community members sharing items"
- [Sign Up Now] button

**Footer:**
- About | How It Works | Safety Tips | Contact | Terms | Privacy
- Social media links

Show detailed information about a specific user

### Layout Description:

**Profile Header:**
- Large profile picture (left)
- Name, location, member since (center)
- Stats row: Items listed, Items borrowed, Points, Average rating
- If own profile: [Edit Profile] button
- If other user: [Message] button (Sprint 4)

- Bio text
- Location (general area with map icon)

- Section title: "Items [Name] is Sharing"
- Filter tabs: [All] [Available] [Unavailable]
- Grid of listing cards (same as browse page)
- Each card: photo, title, category, exchange type, condition

**Reviews/Ratings (Sprint 4):**
- Section title: "Reviews ([count])"
- List of recent ratings with comments
- Each review: rater name, stars, comment, date

## WIREFRAME 7: CREATE LISTING FORM

### Purpose:
Form for users to create new item listings

### Layout Description:

**Page Title:**
- H1: "Create New Listing"
- Progress indicator: Step 1 of 2

**Form Sections:**

**Section 1: Basic Information**
- Title* (text input, max 100 chars)
- Description* (textarea, max 500 chars)
- Category* (dropdown)
- Exchange Type* (radio buttons: Lending / Swap / Giveaway)
  - If Swap selected: Show "What are you looking for?" field

**Section 2: Condition**
- Condition Status* (dropdown: Like New, Good, Fair, Well-Used)
- Condition Notes (textarea, optional, max 200 chars)

**Section 3: Photos**
- Upload Photos (drag & drop or button)
- Support up to 3 photos
- Preview uploaded photos with [Remove] option

**Section 4: Tags** (optional)
- Add Tags (chips/comma-separated)
- Suggested tags appear as you type

**Action Buttons:**
- [Cancel] (secondary)
- [Save as Draft] (optional)
- [Publish Listing] (primary)

**Help Text:**
- Character counters
- Example text in placeholders
- Tips for good listings

## RESPONSIVE DESIGN NOTES

All wireframes should adapt to different screen sizes:

**Desktop (1200px+):**
- Full sidebar filters
- 3-4 cards per row
- Horizontal navigation

**Tablet (768px - 1199px):**
- Collapsible sidebar filters
- 2-3 cards per row
- Horizontal navigation

**Mobile (< 768px):**
- Filters as dropdown/modal
- 1 card per row
- Hamburger menu navigation
- Stacked layouts
- Larger tap targets (48x48px minimum)
